25 ### Cache Organization
26 * caches are *write-allocate, write-back*
27 * 8-entry *victim cache*
28 * each cache must maintain a LRU (least recently used) replacement policy
29  * the least recently used item is discarded first
30 * cache lines are marked as "dirty" if the program has written to that cache entry
31 * on a miss the cache's victim cache is checked before going to the next level or main memory
32  * the victim cache is an 8 deep LRU stack
33  * if the line is available in the victim cache it is moved to the top of the victim cache stack
34  * the line being removed from the cache is then swapped with the entry at the top of the victim cache
35 * if the miss is not available in the victim cache
36  * the entry at the bottom of the victim cache is checked to see if it is "dirty"
37  * if the entry is "dirty" then it is *written back* to the next level of the hierarchy
38  * the line is written as "dirty" in the level the line is moved to
39  * that line is then moved to the top of the victim cache and overwritten with the line being removed from the cache
40  * this is the *write-allocate* portion where room is made in the cache for the new line
41  * the new line is then brought into the cache
42 * a write request only causes the line to be "dirty" on the level that sees the write
43 * after a cache miss
44  * the miss penalty is added to the execution time
45  * it is necessary to account for the time of adding a new block into the cache
46  * this is done by adding the hit time of the cache level to the total execution time

48 ### Program Trace Information
49 * the input traces are a sequence of records that contain reference information
50 * format
51  * Reference type: instruction, data read, data write
52  * Address: memory address in hex - up to 64 bits
53  * Number of bytes: the number of bytes referenced by the request
54 * inst. and data references can fall on any byte boundry -> variable size
55 * an instruction or data reference may generate multiple references to the first-level caches
56  * there is a 4-byte bus between the processor and the L1 caches
57  * all references to the L1 caches look like they are aligned on 4-byte boundaries
58  * the L1 cache returns ad 4-byte value
59 * available traces
60  * testing traces: tr1-tr6
61  * testing traces: parts of the production traces
62  * production traces: 6 straces

69 ### Evaluation
70 * Execution time
71  * inst. takes 1 cycle
72  * inst. fetch 1 cycle
73  * data fetch 1 cycle (for each reference)
74  * **note:** memory references are not aligned - multiple cache accesses possible
75  * total execution time = 1 cycle per inst. + simulated time to access memory hierarchy + 1 * num data refs
76 * Cost
77  * L1 cache
78  * $100 per 4KB
79  * $100 for each doubling of a associativity beyond direct=mapped
80  * cost for doubling the associativity is per 4KB (combined costs are multiplied)
81  * L2 cache
82  * $50 per 16KB
83  * $50 per doubling of associativity
84  * Main Memory
85  * $200 to decrease latency by factor of 2
86  * $100 to increase bandwidth by factor of 2
87  * base setup - latency 50 costs $50 + 8-byte bandwidth 25 = $75
88  * program should compute cost based on the memory configuration specified
